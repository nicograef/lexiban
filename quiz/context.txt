Quiz App — Build Context
========================

Goal: Comprehensive quiz app for learning IBAN, Java, Spring Boot, Docker & more
Tech: Plain HTML + CSS + JS (no frameworks, no bundlers)
Location: /workspaces/iban/quiz/

Questions source: /workspaces/iban/quiz/questions.json
- 130 questions (single-choice, 4 options each)
- Rich explanations with **bold**, `code`, paragraphs, and ≈ TS/Node/Go analogies
- Covers: IBAN/SEPA, Java platform, Spring Boot, JPA/Hibernate, Maven, Docker, Testing, DDD, Architecture Decisions

Files:
1. questions.json  — all 130 questions as structured JSON
2. index.html      — minimal HTML shell
3. style.css       — styling (responsive, mobile-first, rich explanation support)
4. app.js          — quiz logic (render, shuffle, check answer, show rich explanation)

Features:
- Show one question at a time
- User selects an answer, clicks "Antwort prüfen"
- Correct/incorrect feedback with rich explanation shown (supports bold, code, paragraphs, analogy markers)
- Navigate to next/previous question
- Shuffle: questions AND options are randomized on every page load
- Keyboard shortcuts: ←/→ navigate, Enter checks answer
- Mobile-friendly: responsive layout, scrollable explanations, touch-optimized buttons
- No score tracking, no state persistence, no storage

JSON Schema (per question):
  { id, question, options: ["…","…","…","…"], answer: 0, explanation }
- "options" is an array of 4 strings (labels A/B/C/D computed at render)
- "answer" is the 0-based index into the options array pointing to the correct answer
- "explanation" supports light-markdown: **bold**, `code`, \n\n (paragraphs), \n (line breaks), ≈ (analogy marker)
- All 130 questions are single-choice

Shuffle behavior:
- On load: Fisher-Yates shuffle of questions array
- Per question: options array is shuffled, answer index is remapped accordingly
- Shuffle state is stored per-question so navigating back shows the same shuffled order
- No toggle — always shuffled

Explanation rendering (renderExplanation() in app.js):
- escapeHtml() first for XSS safety
- \n\n → <p> paragraph breaks
- \n → <br> line breaks
- **bold** → <strong>
- `code` → <code>
- ≈ → styled <span class="analogy"> (purple highlight for TS/Node/Go analogies)

Decisions / Notes:
- German quotation marks „…" cause JSON parse errors → replaced with '…'
- answer was originally an array ["B"] for multi-choice support, simplified to single string "B", then migrated to 0-based index (number)
- options was originally an object {A:"…",B:"…"}, migrated to array for flexibility and shuffle support
- Single-choice: clicking a new option deselects the previous one
- No score tracking, no localStorage, no persistence
- escapeHtml() used to prevent XSS from question text
- Shuffle always on, no toggle button — maximizes learning on repeat runs
- lernfragen.md content is now fully covered by the 130 quiz questions + rich explanations
- decisions.md ADRs referenced in relevant questions (ADR #1, #3, #4, #6, #11, #12, #13, #14)
